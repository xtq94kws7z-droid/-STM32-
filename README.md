===================================================
 STM32F103C8T6 + BME280 + OLED 项目开发总结
===================================================

一、硬件 I2C 不稳定（STM32F103C8T6 I2C 外设缺陷）
---------------------------------------------------
问题：HAL I2C 读写 BME280 时频繁出错，数据随机跳变
方案：改用软件 I2C（Bit-Bang），PB6=SCL, PB7=SDA
      自己控制时序，绕过 STM32F1 的 I2C 硬件 errata


二、湿度显示 270% 溢出
---------------------------------------------------
问题：湿度值显示 ~300000.1%，显然溢出
原因：Bosch 官方公式中 v_x1_u32r 最后需要 >>12 再 /1024
      但代码写了 v_x1_u32r / 1024.0f（直接被 1024 除，没移位）
      或者写了 >> 10 / 1024.0f
方案：v_x1_u32r >> 12，结果 / 1024.0f 得出 %RH
      正确做法：(float)(v_x1_u32r >> 12) / 1024.0f


三、气压显示 258162 hPa
---------------------------------------------------
问题：气压显示 258162.xx hPa，正常应为 ~1013 hPa
原因：Bosch 64-bit 气压公式最终结果 p 的单位是 1/256 Pa
      需 p/25600 才得到 hPa。代码漏了除以 25600（返回原始 p）
方案：return (uint32_t)(p / 256)
      这样 press/100 和 press%100 就能正确显示 hPa.decimal


四、温度显示 99°C（ADC 符号位误判）
---------------------------------------------------
问题：温度偶尔跳成 99°C 或 -144°C
原因：BME280 的原始温度/气压 ADC 是 20-bit 无符号值
      但代码中做了有符号判断：
        if (adc_T > 0x7FFFF) adc_T -= 0x100000;
      当 rT=524688(=0x80110) > 0x80000 时，被错误转成负数
      之前 rT=523088(=0x7FB50) < 0x80000，所以偶尔正常偶尔坏
方案：删掉所有符号位修正代码，20-bit 无符号 ADC 直接作为 int32_t 正数使用


五、单次 I2C 读取 + 本地补偿（关键架构设计）
---------------------------------------------------
问题：BME280_ReadAll 内部读一次 I2C，main.c 又读一次取原始值
      两次读法数据不一致（传感器在 Normal 模式连续更新）
方案：每次循环只读一次 I2C（8字节，0xF7-0xFE）
      然后本地用 Bosch 公式补偿温度/气压/湿度
      三个值来自同一份原始数据，保证一致性


===================================================
 当前状态
===================================================
- 温度：正常（~24°C）
- 湿度：正常（~45%）
- 气压：正常（~1013 hPa）
- 更新周期：每 2 秒刷新一次
- I2C：软件 Bit-Bang，PB6/PB7
- 传感器模式：Normal（连续自动测量）
- 通讯验证：ST-Link V2 + Keil MDK
